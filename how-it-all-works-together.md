# How it all works together

1. <mark style="color:purple;background-color:purple;">**User Query:**</mark> The user submits their question to your server
2. <mark style="color:purple;background-color:purple;">**Tool Discovery:**</mark> Your server needs to know what tools are available to send to Claude
3. <mark style="color:purple;background-color:purple;">**List Tools Exchange:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">Your server asks the MCP client for available tools</mark>
4. <mark style="color:purple;background-color:purple;">**MCP Communication:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">The MCP client sends a</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">`ListToolsRequest`</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">to the MCP server and receives a</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">`ListToolsResult`</mark>
5. <mark style="color:purple;background-color:purple;">**Claude Request:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">Your server sends the user's query plus the available tools to Claude</mark>
6. <mark style="color:purple;background-color:purple;">**Tool Use Decision:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">Claude decides it needs to call a tool to answer the question</mark>
7. <mark style="color:purple;background-color:purple;">**Tool Execution Request:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">Your server asks the MCP client to run the tool Claude specified</mark>
8. <mark style="color:purple;background-color:purple;">**External API Call:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">The MCP client sends a</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">`CallToolRequest`</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">to the MCP server, which makes the actual GitHub API call</mark>
9. <mark style="color:purple;background-color:purple;">**Results Flow Back:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">GitHub responds with repository data, which flows back through the MCP server as a</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">`CallToolResult`</mark>
10. <mark style="color:purple;background-color:purple;">**Tool Result to Claude:**</mark> <mark style="color:purple;background-color:purple;"></mark><mark style="color:purple;background-color:purple;">Your server sends the tool results back to Claude</mark>
11. **Final Response:** Claude formulates a final answer using the repository data
12. **User Gets Answer:** Your server delivers Claude's response back to the user
